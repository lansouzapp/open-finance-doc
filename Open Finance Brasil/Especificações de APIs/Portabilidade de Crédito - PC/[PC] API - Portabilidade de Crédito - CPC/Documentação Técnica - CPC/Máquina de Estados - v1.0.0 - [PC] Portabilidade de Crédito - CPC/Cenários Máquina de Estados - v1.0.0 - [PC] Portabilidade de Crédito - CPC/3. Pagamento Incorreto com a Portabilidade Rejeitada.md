1. [ RECEIVED ] = Momento que credora recebe solicitação de portabilidade de crédito, este estado deve permanecer até o próximo dia útil (D+1);
2. [ PENDING ] = Prazo de até 3 dias úteis para a etapa de contraproposta, situações podem ocorrer neste estado: Cliente cancelar a solicitação de portabilidade(via APP Credora ou Proponente) antes das 10:00 do terceiro dia útil; A Credora envie a contraproposta dentro do período determinado e o cliente a rejeite ou aceite; Tanto a IF Credora quanto o cliente não tiveram ação, nesse caso, o status permanecerá como PENDING ao longo dos 3 dias úteis (Até as 10:00 do terceiro dia). (opção do cenário)
3. [ ACCEPTED_SETTLEMENT_IN_PROGRESS ] - Inicia-se a partir da comunicação da Credora à Proponente que a operação não foi retida. Dessa forma, a Proponente deverá realizar o pagamento no mesmo dia em que o estado foi ativado (A máquina será atualizada no dia seguinte ao pagamento (D+1));
4. [ ACCEPTED_SETTLEMENT_COMPLETED ] - Prazo de até 2 dias úteis no momento que entra neste estado, podendo ser concluída antes caso ocorra a seguinte situação: Caso a IF Credora reconheça o pagamento. Além da validação da credora é possível que a proponente envie a comunicação do pagamento via API do OFB.
5. [ PAYMENT_ISSUE ] - Indica algum tipo de inconsistência no pagamento feito pela IF Proponente identificada pela IF Credora;
6. [ REJECTED ] – IF Credora rejeitou o pedido pois não houve uma ação de ajuste, dentro dos 2 dias úteis, por parte da IF Proponente.
